//Documentacion de uso de versiones en git con github

//configuramos con los siguientes comandos y nombres para que cada cambio
//guardado quede asociado a este nombre y correo, debe ser el mismo que usamos en github
//La variable global especifica que es para todos los proyectos

git config --global user.name -----
git config --global user.email -----


//Con el siguiente comando verificamos que las credenciales esten correctas
//deberia aparecernos nuestras credenciales

git config --global --list 


//Conectamos nuestro proyecto local con github 
//remoto algo externo add agregar y origin 

                 //(Aqui va el link del repositorio)
git remote add origin -------------------------


//Convierte y crea tu carpeta local en una .git para que ahora 
//puedas guardar versiones, detectar cambios y manejar historial

git init

//agregamos o selecionamos TODOS para preaparalos
//Como si los metieramos a una caja

git add .

//lo mismo pero agregamos el archivo especifico

git add -----

//Guardamos una version del proyecto Y agregamos un mensaje con ""
//Solo se agregara lo selecionado en el git add

git commit -m "------"

//Con branch creamos ramas independientes de nuestros proyectos Por lo general se usa main,
//aunque se le puede poner cualquier nombres, mas se debe evitar los espacios y caracteres especiales

git branch -M -----

//Subimos nuestro codigo con push y le damos el origen 
//de la rama que creamos o selecionamos es este caso main

git push -u origin main 



//buenas practicas
//push=enviar, origin=a donde, main=que rama, -u=guardar conexion para el futuro

Carpeta → (git init)
Archivos → (git add)
Versión → (git commit)
Rama → (git branch)
Internet → (git remote)
Subir → (git push)

init → activar Git
add → preparar cambios
commit → guardar versión
branch → nombrar tu historia
remote → conectar a GitHub
push → subir código

feat: → nueva funcionalidad
fix: → corrección de errores
docs: → documentación
style: → formato (espacios, comas, etc.)
refactor: → mejoras internas sin cambiar comportamiento

//RAMAS o BRANCH 
//Las ramas son como si hicieramos otras carpetas con una copia de nuestro trabajo
//el cual es totalmente independiente y si es el caso podemos experimentar


//Con este comando cambiamos de rama 

git checkout nombre-rama


//Con este comando cambiamos de rama y creamos la rama

git checkout -b nueva-rama
git switch -c feature-login


//traer los cambios desde GitHub

git pull origin main

//Los - en Git no son comandos Son “modificadores” del comando principal

//Git abre un editor de texto para escribir el mensaje.

-m = message

//“recuerda esta rama y este remoto como predeterminados”

-u = --set-upstream

//Después de usarlo una vez, puedes hacer:

git push

//verificacion de historial local

git log --oneline --decorate -5


//con este vemos las ramas creadas en el repositorio

git branch


//Confirmamos a que remoto este haciendo push

git remote -v


//Ver ramas 

git branch -a

//Ver commits existentes

git reflog
